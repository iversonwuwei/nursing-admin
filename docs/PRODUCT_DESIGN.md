## 1. 系统定位与设计理念

### 1.1 系统定位

**核心定位：** 养老院运营管理数据中台 — 一套系统打通管理、护理、家属三方，实现老人安全、员工高效、家属安心的三位一体解决方案。

### 1.2 设计理念

| 维度 | 原则 |
|------|------|
| **温暖感** | 界面不像政府系统，用米色/浅绿/浅蓝传递安心、舒适 |
| **老年友好** | 大字号（≥16px正文）、高对比度、大按钮（≥44px触控区）、操作步骤≤3步 |
| **数据驱动** | 所有业务动作沉淀为数据，数据反哺业务决策 |
| **实时联动** | 三端（管理后台/员工APP/家属APP）数据秒级同步 |
| **安全优先** | 跌倒/健康异常/设备故障第一时间报警+责任人直达 |

### 1.3 用户角色

| 角色 | 设备 | 核心诉求 |
|------|------|---------|
| 养老院管理人员 | PC Web | 运营全局总览、入住率、成本分析、人员排班 |
| 护理人员 | 手机 APP | 任务清单、服务打卡、报警响应、交接班 |
| 老人 | 床头屏/手环 | 呼叫求助、健康数据查看、娱乐 |
| 家属 | 手机 APP | 老人状态查看、预约探视、远程视频、账单 |

```
设备数据（MQTT） ──▶ IoT Hub ──▶ 消息队列（Kafka） ──▶ 流处理（Flink） ──▶ 实时大屏
                              │
                              └──▶ 数据仓库（ClickHouse）──▶ BI 分析（Metabase）

用户操作 ──▶ API Gateway ──▶ 业务服务 ──▶ PostgreSQL
                                              │
                                              └──▶ CDC（Debezium）──▶ 数据仓库
```

   - 填写处理记录（文字+照片）

### 9.2 数据隔离策略

| 隔离级别 | 方案 | 适用场景 |
|---------|------|---------|
| **数据库隔离** | 每租户独立数据库 | 高安全要求（大型连锁） |
| **Schema 隔离** | 同库不同 Schema | 中型规模（10-50个园区） |
| **行级隔离** | tenant_id 字段过滤 | 小型 SaaS（共享数据库） |

**推荐：** Schema 隔离 + 行级过滤双保险，兼顾性能与安全。
